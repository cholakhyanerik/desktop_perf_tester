# Desktop Performance Tester 🚀

Автоматизированный кроссплатформенный инструмент на Rust для сравнительного нагрузочного тестирования двух десктопных приложений. Тестер параллельно запускает два приложения, собирает системные метрики их работы с частотой 1 раз в секунду и генерирует наглядные результаты, выявляя победителя по использованию системных ресурсов.

## Основные возможности
- **Кроссплатформенность:** Работает на Windows, macOS и Linux из коробки.
- **Мониторинг ресурсов:** Собирает детальную информацию о потреблении CPU, RAM, Disk I/O, Network I/O и GPU (через `nvidia-smi`, если доступно).
- **Автозавершение:** Грамотно контролирует жизненный цикл тестов. Тест автоматически и безопасно завершается, как только вы закроете любое из двух тестируемых приложений вручную или программно.
- **Генерация отчетов:** Автоматически создает детализированный `comparison.md` (текстовый отчет с указанием победителей в категориях), `full_report.json` с сырыми данными для системной интеграции и 6 наглядных `.png` графиков с отрисовкой линий тренда и легендой.
- **История запусков:** Все отчеты сохраняются не только в текущую папку `reports/`, но и архивируются в папке `report_history/` с отметкой точного времени запуска.

## Требования к системе
- Установленный **Rust** и **Cargo** (через [rustup](https://rustup.rs/)).
- *(Опционально для сбора GPU)* Утилита `nvidia-smi` в системном пути.

## Настройка
1. Склонируйте репозиторий проекта на ваш компьютер.
2. В корневой директории проекта создайте файл `.env` (он добавлен в `.gitignore` и не улетит в публичный репозиторий).
3. Добавьте в файл полные абсолютные пути к тестируемым приложениям. Убедитесь, что для macOS вы указываете путь до исполняемого файла внутри файла `.app` (например: `/Applications/Safari.app/Contents/MacOS/Safari`), а не просто папку:

   ```env
   APP_1_PATH=C:/Users/chola/Downloads/app-v1.exe
   APP_2_PATH=C:/Users/chola/Downloads/app-v2.exe
   ```

## Запуск автотестирования

Для запуска тестирования просто выполните команду в консоли:
```bash
cargo run --release
```

**Процесс тестирования:**
1. Приложения из путей `APP_1_PATH` и `APP_2_PATH` будут запущены автоматически.
2. В терминале отобразится надпись `👀 Monitoring started...`. Выполняйте необходимые вам действия со сравниваемыми приложениями на экране.
3. Программа будет собирать метрики. Когда вы будете готовы закончить тест, просто **закройте одно из приложений** стандартным крестиком или шорткатом.
4. Второе приложение будет закрыто тестером автоматически, и начнется генерация отчетов.

## Где смотреть результаты?
Результаты текущего запуска сохраняются в директорию `reports/`. 
Вы найдете там:
- `comparison.md` — Итоговое текстовое сравнение с указанием приложения-победителя.
- `cpu_comparison.png`, `ram_comparison.png` и др. — Графики метрик с двумя линиями (App 1 / App 2).
- `full_report.json` — Все замеры в формате JSON.

Предыдущие запуски можно найти в папке `report_history/РАНДОМНЫЙ_ID_ЗАПУСКА/`.

## По поводу системных метрик сети и GPU
Так как детальный срез сетевой активности и нагрузки видеоядра на каждую конкретную программу является сложным для кроссплатформенной C++ / Rust реализации (без тяжеловесных привязок к ядру типа WMI/eBPF/NVML), сбор с метрик по GPU и сети реализован как **общесистемный**. То есть `network_comparison.png` отражает общую тенденцию сетевого трафика в момент работы приложения.
